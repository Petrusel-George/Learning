# Start

## 1. Basic Concepts and Terminology

- **Packet:** A small unit of data that is transmitted over the internet.
- **Router:** A device that redirects the packets of data between different networks.
- **IP Address:** A unique identifier assigned for each computer, used to route data to the correct destination.
- **Domain name:** An alias used for IP addresses for human readability.
- **DNS:** The Domain Name System is responsible for translating domain names into IP addresses.
- **HTTP:** The HyperText Transfer Protocol is used to transfer data between a client such as a browser and a server such as a website.
- **HTTPS:** An encrypted version of the HTTP that is used to provide secure communication between a client and a server.
- **SSL/TLS**: The Secure Socket Layer and Transport Layer Security protocols are used to provide secure communication over the internet.

## 2. The role of protocols over the internet

A protocol is a set of rules and standards that define how information is exchanged between devices and systems.

```example
    Internet Protocol (IP)
    Transfer Control Protocol (TCP)
    User Datagram Protocol (UDP)
    Domain Name System (DNS)
```

### How protocols work together:

IP provides the "address labels" on packets → Routers read those labels → Routing protocols determine paths → Routers forward packets along those paths.

### TCP vs UDP (Transport Layer Protocols):

**TCP (Transmission Control Protocol):**

- Reliable and ordered delivery
- Ensures all packets arrive in correct sequence
- Slower but guarantees accuracy
- Used for: Web browsing (HTTP/HTTPS), email, file transfers
- Establishes a connection before sending data (handshake)

**UDP (User Datagram Protocol):**

- Fast but unreliable
- No guarantee all packets arrive or arrive in order
- Lower latency, higher speed
- Used for: Video streaming, online gaming, VoIP, live broadcasts
- No connection setup required (connectionless)

**When to use which:**

- Choose TCP when data accuracy is critical (financial transactions, messages)
- Choose UDP when speed matters more than perfection (real-time video, gaming)

## 3. Data transfer over the internet

### The Request/Response Flow:

1. **Client initiates request:** Your browser sends a request to a server (e.g., requesting www.google.com)

2. **DNS lookup:** Your device queries DNS servers to translate the domain name into an IP address

3. **Connection established:** For HTTP/HTTPS, TCP establishes a connection with the server (3-way handshake)

4. **Data sent in packets:** Your data is broken into small packets that are sent from your device to a router

5. **Routing through networks:** Each router examines the packet header and forwards it to the next router along the path to the destination. The route can dynamically change based on network traffic to ensure efficiency

6. **Server receives packets:** Packets arrive at the destination server (not necessarily in order over the internet)

7. **TCP reassembly:** TCP ensures all packets have arrived and reassembles them in the correct order

8. **Server processes request:** The server processes the request and sends a response back through the same process

9. **Client receives response:** Your browser receives the response data and renders the webpage

### Ports (Addressing within an IP):

A single computer can run multiple services simultaneously using **ports**:

- **Port 80:** HTTP (unencrypted web traffic)
- **Port 443:** HTTPS (encrypted web traffic)
- **Port 25/587:** SMTP (email sending)
- **Port 110/995:** POP3 (email receiving)
- **Port 21:** FTP (file transfer)

So a complete address is: `IP_Address:Port_Number` (e.g., 142.199.10.78:443)

### Packet Structure:

Each packet contains:

- **Header:** Source IP, destination IP, port numbers, protocol type, sequence number
- **Payload:** The actual data being transmitted
- **Footer:** Error-checking information (checksum)

This structure allows routers to know where to send the packet and the destination computer to know which application (port) should receive it.

## 4. TCP/IP: Understanding the Foundation for Application Development

TCP/IP (Transmission Control Protocol/Internet Protocol) is the underlying communication protocol used by most internet-based applications and services. It provides a reliable, ordered, and error-checked delivery of data between applications running on different devices.

### Key Concepts for Building Applications:

**Sockets:**
A socket is a combination of an IP address and a port number, representing a specific endpoint for communication. Sockets are used to establish connections between devices and transfer data between applications. Think of a socket as a "mailbox address" that uniquely identifies where an application is listening for incoming data.

Example: A web server listening on `192.168.1.100:443` creates a socket that waits for incoming HTTPS connections.

**Connections:**
A connection is established between two sockets when two devices want to communicate with each other. During the connection establishment process, the devices negotiate various parameters such as:

- **Maximum segment size:** Determines the largest chunk of data that can be sent in a single segment
- **Window size:** Determines how much data can be sent before waiting for acknowledgment

These negotiations ensure reliable and efficient data transmission.

**Data Transfer in Segments:**
Once a connection is established, data can be transferred between applications. Data is typically transmitted in segments, with each segment containing:

- **Sequence number:** Ensures packets arrive in the correct order
- **Acknowledgment number:** Confirms receipt of data
- **Other metadata:** Checksums, flags, and timing information

This ensures reliable delivery even if some packets are lost or arrive out of order.

**Common Protocols Built on TCP/IP:**

- **HTTP/HTTPS:** Web browsing (ports 80/443)
- **FTP:** File Transfer Protocol (port 21)
- **SMTP:** Simple Mail Transfer Protocol for email sending (port 25/587)
- **POP3:** Post Office Protocol for email retrieval (port 110)
- **DNS:** Domain Name System for translating domain names (port 53)

### Implications for Application Development:

When building internet-based applications, you must understand:

1. **Port selection** — Choose appropriate ports for your service and ensure they're not in use
2. **Socket management** — Properly create, maintain, and close sockets
3. **Connection lifecycle** — Handle connection establishment, data transfer, and graceful closure
4. **Error handling** — Account for lost packets, timeouts, and network failures
5. **Protocol compliance** — Ensure your application follows the standards (HTTP, SMTP, etc.)

Understanding these concepts is essential for building effective, scalable, and secure internet-based applications and services.
